Descripción funcional

  1. Definir caso — data/casos/{id}/caso.json + medición baseline
  2. Aportar buffer — JSON en data/buffers/
  3. Cribar — medidor cribar archivo.json --caso {id}
  4. Commit — medidor commit archivo.json --caso {id}
  5. Sincronizar catálogo — medidor catalog sync
  6. Publicar — medidor build --target all (HTML prensa + paquetes ZIP en public/prensa/downloads/)

El caso foco es inmutable. Los buffers son apéndices; no sobrescriben anteriores.

Los visitantes del centro de datos pueden descargar cada medición o el caso completo como ZIP; los botones enlazan a esos archivos estáticos generados en el paso de publicación.
